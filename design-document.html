Design System Overview

This is a neo-brutalist design system featuring toxic orange as the primary accent color. The aesthetic is characterized by thick black borders, hard offset shadows (not drop shadows), flat colors with no gradients, and chunky bold typography. The system prioritizes playful precision—energetic and snappy while maintaining strict geometric layouts.

Key Aesthetic Principles:

  • Neo-Brutalism: Thick black borders, offset shadows (6px blocks), no rounded corners
  • Color: Toxic orange (#FF6B00) primary, near-black backgrounds, concrete grays, stark white text
  • Typography: Chunky sans-serif (Space Grotesk), bold weights, tight spacing, uppercase labels
  • Interaction: Snappy button feedback (shadow collapse on press), smooth micro-animations
  • Layout: Strict geometric grids, minimal padding within elements, breathing room between sections

Implementation Notes

Chat Landing Page Structure:

The hero chat interface should take up the full viewport with edges fading to dark. Here's the recommended structure:

<div class="chat-container">
  <div class="chat-messages">
    !-- Message bubbles here --
  </div>
  <div class="chat-input-container">
    !-- Input field + send button --
  </div>
</div>

Key Interaction Patterns:

  • Button Press: Shadow collapses from 6px → 3px on hover, fully collapses on active
  • Card Hover: Slight translate (3px) with shadow reduction for tactile feedback
  • Input Focus: Border changes to orange, shadow changes to orange offset
  • Message Entry: Slide up animation (translateY + opacity fade-in)
  • Mobile Nav: Slide down from top with opacity transition

Responsive Breakpoints:

  • Desktop: Max width 1400px for content, centered with breathing room
  • Tablet: 768px breakpoint - timeline collapses to left, nav becomes hamburger
  • Mobile: Card grid becomes single column, profile stacks vertically, reduced spacing

Performance Optimizations:

  • Use CSS transforms for animations (GPU-accelerated) instead of position changes
  • Lazy load case study images below the fold
  • Minimize chat message re-renders with proper React keys or vanilla JS optimization
  • Consider virtual scrolling for very long chat histories
